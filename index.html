// Process

How it works

Three steps from spotted bug to verified report.

1

Spot a bug

Find issues: broken flows, missing redirects, typos, performance degradation, or security concerns on any public platform.

2

Submit a report

Use the GitHub issue template, the web form below, or call the BLT-API for anonymous submissions. Attach screenshots, logs, and severity.

3

Earn & build trust

Climb the leaderboard, earn community rewards, and build a reputation as a responsible security researcher.

// BLT API

Submit via CLI or API

Power users can report bugs programmatically using the BLT-API.

blt — bash — 80×24
# Install BLT CLI
$ pip install owasp-blt
 
# Submit a bug anonymously
$ blt report --url "https://example.com/broken" \
       --type "functional" --anon
 
Submitting report...
Verifying domain... ✓
Report #19 filed successfully. Thank you!
 
$
